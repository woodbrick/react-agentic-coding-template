# 🧠 上下文工程配置指南

> 智能上下文管理系统的核心配置规范，为AI助手提供工程化的上下文支持

## 🎯 目录结构定义

### checklists/ - 检查清单库
- **功能**: 代码审查、发布前检查、质量门禁的可执行检查项
- **内容**: 验收标准、检查清单
- **禁止**: 流程说明、设计原则、技术解释

### examples/ - 最佳实践示例库
- **功能**: 提供可直接复用的完整代码模板和配置示例
- **内容**: 测试验证的代码示例、配置模板、工作流程
- **禁止**: 理论说明、设计决策、规范文档

### knowledge/ - 技术指南和知识库
- **功能**: 框架、库、工具的使用说明和操作指南
- **内容**: 技术文档、操作手册
- **禁止**: 项目专属编码规范、架构决策

### prds/ - 产品需求文档档案
- **功能**: 记录产品需求和验收标准
- **内容**: 业务目标、用户场景、功能清单、验收标准
- **禁止**: 实现细节、代码、设计讨论

### rules/ - 规范和标准文档
- **功能**: 定义项目的技术规范和质量标准
- **内容**: 设计原则、架构决策、质量标准
- **禁止**: 具体实现代码、模板

### stories/ - 用户故事档案
- **功能**: 记录用户故事和实现方案
- **内容**: 用户故事、任务拆分、业务价值、实现路径、验收标准
- **禁止**: 代码、配置文件、技术实现细节

### tasks/ - 任务执行记录
- **功能**: 记录已完成的任务和变更
- **内容**: 接口变更记录、任务执行报告
- **禁止**: 未执行的待办事项、草稿、设计讨论

### templates/ - 文档模板库
- **功能**: 提供标准化的文档和配置模板
- **内容**: 文档模板、配置模板
- **禁止**: 项目特定占位符、环境相关配置

### traps/ - 常见陷阱警示
- **功能**: 收集和分享历史问题和解决方案
- **内容**: 问题描述、根因分析、解决方案
- **禁止**: 未验证的猜测、主观意见
