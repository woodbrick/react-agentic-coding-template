# 🛠️ Claude Code Skill 创建指南

> 基于项目真实案例的技能开发指南，让你快速创建有用的自动化助手

## 🎨 一复制就能用的模板

直接复制下面的模板，修改内容即可快速创建你的第一个Skill：

```yaml
---
name: "技能名称"
description: "功能描述。当[场景]时使用。"
allowed-tools: [Read, Grep, Glob]  # 可选工具限制
---

# 技能名称

## 操作步骤
1. 第一步
2. 第二步

## 注意事项
- ✅ 推荐做法
- ❌ 避免做法
```

**下一步**：
1. 创建目录：`mkdir -p .claude/skills/你的技能名称`
2. 新建文件：`touch .claude/skills/你的技能名称/SKILL.md`
3. 粘贴模板并修改内容
4. 在聊天中测试："我需要[你的技能描述]"

## 🤔 Skill 是什么？

Skill 是让 Claude 变得更强大的模块化工具包。它能让 Claude 自动识别特定场景并执行预设操作。

**与普通命令的区别**：
- **普通命令**：需要你显式输入 `/command` 来触发
- **Skill**：Claude 根据对话内容自动判断何时使用

**核心优势**：
- **智能触发**：基于你的需求自动激活
- **专注执行**：只做它最擅长的事情
- **团队共享**：通过 Git 让整个团队受益

## 🚀 5分钟创建你的第一个 Skill

### 1. 选择技能类型
根据使用范围选择：
- **项目技能**：`.claude/skills/技能名称/` - 团队共享
- **个人技能**：`~/.claude/skills/技能名称/` - 个人专用

### 2. 创建目录结构
```bash
# 项目技能
mkdir -p .claude/skills/my-skill-name
```

推荐结构：
```
my-skill/
├── SKILL.md          # 核心文件
├── reference.md      # 参考文档
└── examples.md       # 使用示例
```

### 3. 编写 SKILL.md 文件
这是技能的核心，包含：
- **基本信息**：名称、描述、权限设置
- **操作指南**：具体的执行步骤
- **使用示例**：实际应用场景

**示例格式**：
```yaml
---
name: "技能名称"
description: "功能描述。当[场景]时使用。"
allowed-tools: [Read, Grep, Glob]  # 可选工具限制
---

# 技能名称

## 操作步骤
1. 第一步
2. 第二步

## 注意事项
- ✅ 推荐做法
- ❌ 避免做法
```

## ✍️ 写好描述的黄金法则

**好的描述 = 功能 + 场景**

✅ 好的例子：
```yaml
description: "分析Excel表格，创建数据透视表和图表。当处理Excel文件、电子表格或分析.xlsx格式的表格数据时使用。"
```

❌ 不好的例子：
```yaml
description: "处理文档"  # 太模糊，Claude不知道何时该用
```

## 🧪 验证你的技能

**3步验证法**：
1. **检查文件**：`ls .claude/skills/你的技能/SKILL.md`
2. **验证格式**：`cat .claude/skills/你的技能/SKILL.md | head -n 10`
3. **测试触发**：在聊天中提出相关问题，看Claude是否会自动使用

## 🔧 常见错误及解决

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Skill不触发 | 描述太模糊 | 明确说明使用场景和关键词 |
| Skill加载失败 | YAML格式错误 | 检查 --- 是否成对，使用空格而非制表符 |
| 权限不足 | 工具限制过多 | 仅限制必要工具，确保能完成任务 |

## 📊 实战案例：mock-generator 技能

看一个真实项目的优秀案例：

```yaml
---
name: Mock数据生成
description: "为BigFish框架生成符合规范的Mock数据，当需要为前端开发创建API模拟数据时使用，特别是处理/mock目录下的数据生成任务。"
allowed-tools: [Read, Grep, Glob, Write, Edit, Bash]
---
```

**为什么这个技能优秀？**
- **触发清晰**：明确指出使用场景（前端Mock数据生成）
- **功能专注**：只做一件事，而且做得很好
- **权限合理**：只开放必要的文件操作权限
- **标准一致**：遵循项目Mock数据规范

## 🚀 最佳实践总结

**创建优秀Skill的5个要点**：

1. **专注** - 每个技能只解决一个问题
2. **明确** - 描述中包含清晰的触发条件
3. **简单** - 操作步骤要能1步完成，不复杂
4. **可验证** - 提供明确的检查方法
5. **可共享** - 放在项目中，让团队一起受益
